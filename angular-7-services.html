Services

Angular

”Service is a broad category encompassing any value, function, or feature that an app needs. A service is typically a class with a narrow, well-defined purpose. It should do something specific and do it well.” - Angular

Adding a service

  • Needs to be in the module as a provider
  • Functionality not effecting the view directly
  • Storing global state, public and private
  • Can be injected

Services can:

  • Share information among classes that don’t know each other.
  • Be injected into multiple places.
  • Be set in Providers: [UserService] in component creates a new instance if the service. - Instead of the root
